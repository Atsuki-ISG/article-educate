# 執筆ルール

このドキュメントは、articles配下の教育記事を執筆する際の文体、構成、表現の指針をまとめたものです。

**対象読者：** 地域の住民（保護者、地域の方々）
**執筆者：** 教育委員会職員

## 文体の原則

### 1. 基本トーン：親しみやすい「ですます」調

#### ✅ 推奨される表現

**語りかけるような文体**
- 「〜してみてください」「〜しましょう」で読者を導く
- 「〜んです」「〜なんです」で親近感を出す
- 「〜でしょうか？」で読者に問いかける

```markdown
良い例：
- 「お子さんの様子、気になることはありませんか？」
- 「これが大切なポイントなのですが、**子どもの話を「聴く」ことが、信頼関係の第一歩**なんです。」
- 「最初から完璧じゃなくても大丈夫です。」
```

**励ましと共感の表現**
- 「大丈夫です」「心配ありません」
- 「一緒に考えていきましょう」
- 「少しずつ〜していけば大丈夫です」
- 「悩んでいるのは、お子さんのことを大切に思っているからこそです」

```markdown
良い例：
- 「最初からうまくいかなくても大丈夫。少しずつ関わり方を工夫していけば、必ず変化が見えてきますよ。」
- 「長い記事でしたが、最後までお読みいただきありがとうございます。」
```

**口語的な表現の活用**
- 「〜ってなんだろう？」「〜って何？」
- 「〜じゃありません」「〜じゃなく」
- 「ぐっと」「ちょっと」「とても」などの副詞
- 「ふんわり」「ざっくり」などの柔らかい表現

```markdown
良い例：
- 「心の癒しってなんだろう？」
- 「お子さんの気持ち、ざっくりでいいので聞いてみてください。」
- 「声かけを少し変えるだけで、ぐっと話しやすくなります」
```

#### ❌ 避けるべき表現

**硬い・堅苦しい表現**
```markdown
悪い例：
- 「〜である」「〜だ」などの断定調
- 「〜すべきである」「〜しなければならない」
- 「本稿では」「以下に示す」などの論文調
- 「〜に関して」「〜について述べる」などの形式的な表現
- 「児童生徒」（→「子どもたち」「お子さん」に言い換え）
```

**距離を感じさせる表現**
```markdown
悪い例：
- 三人称での説明（「保護者は〜」ではなく「皆さんは〜」「私たちは〜」）
- 受動態の多用（「〜される」ではなく「〜します」）
- 行政文書のような無機質な表現
```

### 2. 専門用語の扱い方

#### 必ず平易な説明を添える

教育用語や行政用語を使う場合は、一般の方にも分かるように説明を加えます。

```markdown
良い例：
- 「生徒指導提要（学校での子どもへの関わり方をまとめた国の指針）」
- 「自己肯定感（自分のことを大切に思える気持ち）」
- 「レジリエンス（心の回復力）」

悪い例：
- 「生徒指導提要に基づき〜」（説明なしで専門用語を使用）
- 「発達支持的生徒指導を推進する」（専門用語のみ）
```

#### 日本語の平易な言葉を優先

可能な限り、専門用語を日常的な言葉で言い換えます。

```markdown
良い例：
- 「居場所づくり」（心理的安全性の確保、とも言いますが）
- 「話を聴く」（傾聴）
- 「見守る」（観察・支援）

カッコ書きで専門用語を併記：
- 「心の回復力（レジリエンス）」
- 「自分で決める力（自己決定能力）」
```

### 3. 文章の長さとリズム

#### 1文は短く、簡潔に

- 1文は2〜3行以内が目安
- 複雑な内容は箇条書きに分解
- 長い説明は段落を分ける

```markdown
良い例：
「子どもたちの心は、日々さまざまなストレスにさらされています。学校のこと、友だちのこと、家庭のこと。私たち大人が思っている以上に、子どもたちは多くのことを抱えているんです。」

（短い文で区切り、読みやすくする）
```

#### 疑問形で興味を引く

セクションの冒頭や小見出しで疑問形を使い、読者の興味を引きます。

**重要：疑問文の語尾は必ず「？」マークをつける**
- 「〜でしょうか？」「〜ませんか？」「〜ですか？」
- 「〜でしょうか。」「〜ませんか。」は ❌

```markdown
良い例：
- 「お子さんの様子、最近どうですか？」
- 「なぜ今、心の教育が大切なの？」
- 「こんなとき、どう声をかけたらいいでしょうか？」
- 「お子さんが「学校に行きたくない」と言ったこと、ありませんか？」

悪い例：
- 「お子さんの様子はいかがでしょうか。」（疑問文なのに「。」）
- 「経験はありませんか。」（疑問文なのに「。」）
```

## 構成の原則

### 1. 明確な階層構造

#### 長い記事は大きな区切りを設ける

長い記事は「第1部」「第2部」や「はじめに」「本編」「まとめ」のように大きく区切ります。

#### 各セクションに「なぜその話をするのか」のリード文を追加

各セクション（特に大きなセクション）の冒頭には、以下を含むリード文を追加します：

1. **なぜその話をするのか**：読者がそのセクションを読む動機を与える
2. **体験談・エピソード（適度に）**：現場での経験や気づきを交えて親近感を出す
3. **次の展開の予告**：このセクションで何を学ぶかを明示

```markdown
良い例：

## 家庭でできる「心の居場所づくり」

学校での取り組みをお伝えしましたが、ここからは「家庭でできること」についてお話しします。実は、ある保護者の方から「学校でやっていることを、家でもやってみたい」というお声をいただいたことがきっかけで、このセクションを書くことにしました。

ここでは、日常の中で実践できる3つのポイントをご紹介します。
```

**リード文のバランス：**
- すべてのセクションにエピソードを入れる必要はない（かえって冗長になる）
- 読者が共感しやすい場面、または重要な転換点で使う
- 自然な流れを保つことを優先する

#### 各セクションは「ポイント」「コツ」で要点明示

読者が何を学ぶかを最初に明示します。

```markdown
### お子さんの話を「聴く」コツ

**ポイント：** アドバイスより先に、まず「そうだったんだね」と受け止めましょう

#### 聴くときの3つのコツ

**コツ1: 手を止めて、目を見る**
**コツ2: 「うん、うん」とうなずく**
**コツ3: 最後まで聞いてから話す**
```

### 2. 具体例を豊富に使う

#### エピソードで伝える

抽象的な説明ではなく、具体的な場面を描写します。

```markdown
良い例：
**【エピソード】ある朝の出来事**

「行ってきます」の声がいつもより小さかったAさん。お母さんは「どうしたの？」と聞きたい気持ちをぐっとこらえて、「いってらっしゃい、気をつけてね」と送り出しました。

その日の夕方、Aさんの方から「今日ね、ちょっと嫌なことがあって...」と話し始めたそうです。
```

#### 良い例と悪い例の対比

❌と✅を使って視覚的に分かりやすく。

```markdown
❌ **こんな声かけしていませんか？**

「なんで言わなかったの？」
「早く言ってくれればよかったのに」

これだと、お子さんは「怒られた」と感じてしまうかもしれません。

✅ **こんな声かけに変えてみましょう**

「話してくれてありがとう」
「それは辛かったね」

まず受け止めることで、「話してよかった」と思ってもらえます。
```

### 3. 視覚的な整理

#### 箇条書きと番号付きリストを活用

情報は箇条書きや番号付きリストで整理します。

```markdown
**お子さんのサインに気づくポイント：**
- 食欲の変化（急に食べなくなった、食べすぎるなど）
- 睡眠の変化（寝つきが悪い、朝起きられないなど）
- 表情の変化（笑顔が減った、ぼんやりしているなど）

**相談できる場所：**
1. 担任の先生
2. スクールカウンセラー
3. 教育相談窓口（電話番号: XXX-XXXX）
```

#### 表で情報を比較

対比や比較は表形式で見やすく。

```markdown
| 気になる行動 | 考えられる背景 | 対応のヒント |
|----------|----------|----------|
| 朝、起きられない | 睡眠不足、ストレス | まず体調を確認、無理に起こさない |
| 食欲がない | 緊張、不安 | 好きなものを少量から、会話を大切に |
```

### 4. まとめセクション

#### 各パートの最後に要点を復習

読者が学んだことを整理できるように、まとめを入れます。

```markdown
## まとめ

### この記事でお伝えしたこと

最後までお読みいただき、ありがとうございます。

この記事では、お子さんの心に寄り添うために大切なことをお伝えしました：

**家庭でできること**
- お子さんの話を「聴く」姿勢
- 「そうだったんだね」と受け止める声かけ
- 一緒に過ごす時間を大切にする

**困ったときの相談先**
- 学校（担任、スクールカウンセラー）
- 教育相談窓口
```

#### 励ましの言葉で締めくくる

記事の最後は、読者を励まし、安心感を与えます。

```markdown
### 最後に

子育てに正解はありません。悩むこと、迷うことがあって当然です。

大切なのは、お子さんのことを想う気持ち。その気持ちがあれば、きっと大丈夫です。

困ったときは、一人で抱え込まず、ぜひ私たちにご相談ください。学校も、教育委員会も、皆さんと一緒にお子さんの成長を見守っていきたいと思っています。
```

## 表現の指針

### 1. 読者との距離を縮める表現

#### エピソードで親近感を出す

適度に現場での経験や保護者の声を交えることで、読者との距離が縮まります。

**使いどころ：**
- 新しいセクションのリード文
- 重要なポイントを強調したいとき
- よくある悩みや困りごとを説明するとき
- 読者が「本当に効果あるの？」と疑問に思いそうなとき

```markdown
良い例：
- 「ある保護者の方から「うちの子、最近何も話してくれなくて...」というご相談をいただきました。」
- 「実は、学校の先生たちも最初から上手にできたわけではありません。試行錯誤を重ねて、少しずつコツをつかんでいったんです。」
- 「「聴く」って簡単そうで難しいですよね。私たちも日々、子どもたちと向き合いながら学んでいます。」
```

**注意点：**
- すべてのセクションにエピソードを入れる必要はない
- エピソードが主役にならないよう、簡潔に（2〜3文程度）
- 個人が特定されないよう配慮する
- エピソード→本題への自然な流れを作る

#### 二人称で語りかける

「保護者」「住民」ではなく「皆さん」「あなた」を使います。

```markdown
良い例：
- 「皆さんのご家庭ではいかがですか？」
- 「お子さんと一緒に、試してみてください」
- 「私たちと一緒に、子どもたちを見守っていきましょう」

悪い例：
- 「保護者は〜すべきである」
- 「各家庭において〜」
```

#### 共感と寄り添い

読者の気持ちに寄り添う表現を使います。

```markdown
良い例：
- 「お子さんのこと、心配になりますよね。」（共感）
- 「どう声をかけたらいいか、迷うことありませんか？」（読者の悩みを代弁）
- 「うまくいかない日があっても、大丈夫です。」（励まし）
```

### 2. 公的機関としての信頼性

#### 根拠を示す

必要に応じて、公的な指針や資料を引用します。

```markdown
良い例：
- 「文部科学省の「生徒指導提要」（令和4年改訂）では、〜と示されています。」
- 「〇〇市の調査によると、〜という結果が出ています。」
```

#### 相談先を明記する

困ったときの相談先を具体的に示します。

```markdown
良い例：
**ご相談はこちらまで**
- 〇〇市教育相談窓口：XXX-XXXX（平日9:00〜17:00）
- 各学校のスクールカウンセラー
- 担任の先生
```

### 3. 段階的な説明

#### 「まず〜」「次に〜」「最後に〜」で順序を明示

```markdown
**お子さんの話を聴くとき**

**1. まず、手を止める**
スマホを置いて、お子さんの方を向きましょう。

**2. 次に、うなずきながら聴く**
「うん、うん」「そうなんだ」と相づちを打ちましょう。

**3. 最後に、気持ちを受け止める**
「それは嫌だったね」「頑張ったんだね」と声をかけましょう。
```

### 4. 強調と注目

#### 太字で重要ポイントを強調

```markdown
そして、ここが大切なポイントなのですが、**子どもは「話を聞いてもらえた」という経験が、心の安定につながる**んです。
```

#### 絵文字は使わない

親しみやすさは文体で表現し、絵文字は使用しません。

```markdown
❌ 悪い例：
「一緒に頑張りましょう！💪」

✅ 良い例：
「一緒に、少しずつ取り組んでいきましょう。」
```

### 5. 否定的な表現を避ける

#### 「できない」より「〜する方法」

```markdown
悪い例：
「これでは子どもは話してくれません」

良い例：
「こう声をかけると、話しやすくなります」
```

#### 「難しい」より「一緒にやってみよう」

```markdown
悪い例：
「子どもの気持ちを理解するのは難しいです」

良い例：
「子どもの気持ち、一緒に考えてみましょう」
```

#### 保護者を責めない

```markdown
悪い例：
「こういう対応は避けてください」
「これでは子どもが傷つきます」

良い例：
「こんな工夫をすると、もっと伝わりやすくなります」
「お子さんにとって、より安心できる声かけをご紹介します」
```

## 教育記事特有の注意点

### 1. 公文書との使い分け

教育委員会からの発信には、公文書（通知、お知らせ）と広報記事があります。この執筆ルールは**広報記事**に適用します。

| 種類 | 文体 | 目的 |
|------|------|------|
| 公文書 | 「である」調、正式な表現 | 正確な情報伝達 |
| 広報記事 | 「ですます」調、親しみやすい表現 | 理解促進、啓発 |

### 2. 引用・参照のルール

#### 生徒指導提要等の公的文書を引用する場合

```markdown
良い例：
令和4年に改訂された「生徒指導提要」では、生徒指導を「児童生徒が、社会の中で自分らしく生きることができる存在へと、自発的・主体的に成長や発達する過程を支える教育活動」と定義しています。

少し難しい言葉ですが、つまり「子どもたちが自分らしく成長できるよう、学校全体で支えていく取り組み」ということです。
```

#### 参考文献の記載

記事の最後に参考文献を記載します。

```markdown
---

**参考文献**
- 文部科学省「生徒指導提要」（令和4年12月改訂）
- 〇〇市教育振興基本計画
```

### 3. プライバシーへの配慮

エピソードを紹介する際は、個人が特定されないよう配慮します。

```markdown
良い例：
- 「ある小学生のお子さん」「中学生のBさん」
- 「保護者の方からこんなご相談がありました」
- 具体的な学校名、地域名は出さない

悪い例：
- 「〇〇小学校の3年生の男の子」
- 「△△地区にお住まいの保護者から」
```

## 具体的な執筆プロセス

### 1. 構成を先に決める

1. 大まかな目次を作成
2. 各セクションの見出しを決める
3. 各セクションで伝えたいポイントを箇条書き
4. 具体的なエピソードを考える
5. 本文を執筆

### 2. 執筆時のチェックリスト

#### 文体チェック
- [ ] 「ですます」調で統一されているか
- [ ] 読者に語りかける表現を使っているか
- [ ] 専門用語に説明を添えているか
- [ ] 1文が短く、簡潔か
- [ ] 保護者を責めるような表現がないか

#### 構成チェック
- [ ] 見出しの階層は適切か
- [ ] 各セクションに「ポイント」や「コツ」が明示されているか
- [ ] 良い例と悪い例の対比があるか
- [ ] 具体的なエピソードは豊富か
- [ ] まとめセクションがあるか
- [ ] 相談先が明記されているか

#### 表現チェック
- [ ] 励ましや共感の表現があるか
- [ ] 疑問形で興味を引いているか（語尾は「？」）
- [ ] 箇条書きや表で整理されているか
- [ ] 重要なポイントは太字で強調されているか
- [ ] 否定的な表現を避けているか
- [ ] プライバシーに配慮しているか

#### 公的機関としてのチェック
- [ ] 根拠となる資料・指針を適切に引用しているか
- [ ] 参考文献を記載しているか
- [ ] 誤解を招く表現がないか

### 3. 執筆後の見直し

1. 音読してリズムを確認
2. 保護者目線で「分かりやすいか」をチェック
3. 専門用語に説明があるか確認
4. エピソードが適切か確認（プライバシー配慮）
5. まとめで全体を復習できるか確認
6. 相談先の情報が正確か確認

## フィードバック反映のプロセス

記事に対してフィードバックを受けた際は、以下のプロセスで対応します：

### 1. フィードバックの記録と分析

受け取ったフィードバックを具体的に記録します：
- どのような指摘か（文体、構成、表現など）
- 具体例があれば記録する
- 影響範囲を確認する（一部 or 全体）

### 2. 記事への反映

フィードバックに基づいて記事を修正します：
- 該当箇所を特定して修正
- 同様の問題が他にないか確認
- 修正後の整合性をチェック

### 3. 抽象化と学びの抽出

個別のフィードバックから、一般化できる原則を抽出します：

**例：**
```
【フィードバック】
「生徒指導提要との関連付けを入れてほしい」

【抽象化した学び】
- 公的な指針・資料との関連付けは信頼性を高める
- 専門用語は必ず平易な言葉で言い換える
- 引用後に「つまり〜」と噛み砕いた説明を加える

【執筆ルールへの反映】
→「引用・参照のルール」セクションに具体例を追加
```

### 4. CLAUDE.mdへの反映

抽象化した学びを適切なセクションに追加します：
- 既存のルールの拡充 or 新規セクションの追加
- 具体例を必ず含める（良い例・悪い例）
- なぜそのルールが重要かの理由も記載

### 5. 更新履歴への記録

CLAUDE.mdの「更新履歴」セクションに以下を記録します：
- 日付
- 追加・修正した内容の概要
- フィードバック元（記事名など）
